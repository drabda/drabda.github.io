Part 1: What is Programming? 01:37

  • Computers need to be given very specific (exact) instructions to complete a task.
  • Computers are dumb. We just manipulate it to serve our needs.
  • Computers only understand binary (machine code).
  • Imagine your friend only speaks mandarin, and you only speak English. You must find a way to communicate with your friend.
  • This is like computer code: We need a way for us to convey (translate) what we want in a way that the computer understands (interprets) and carries out.
  • Coding in only binary is inconvenient, so we have programming languages that translate from English, and instruct the computer to do tasks in a way it can comprehend.
  • Different languages have 'levels': High level, low level. These specify how similar (or close) they are to machine code. Python is high level, whereas Assembly is low level.
  • The lower the level, the more your code will resemble what the machine can interpret as instructions.
  • Choice of languages usually come down to preference.

Part 5: What are Variables? 20:43

  • A Variable is something that stores information.

    • Can be referenced
    • Can be manipulated
    • Has a Type
    • Has a Name (Simply a name for the variable)
    • Has a Piece of information inside of it
    • Variables contain many types

     

     

  • Primitave Variable Types:

  • Integers are a variable that can store an Integer Value:

    • All whole numbers from: -2, 147, 483, 648 to 2, 147, 483, 648
    • Integer values cannot and will not hold any decimal values

  • Booleans can store a value of either true or false

    • Can only hold these two values

  • Floats and Doubles are both types of floating point data types

    • Float variables can store up to 32 bits of information
    • Double Variables can store up to 64 bits of information
    • Both can store numbers with decimal values

  • String Variables can store strings of letters, usually words and sentences.

  • Valuable for displaying text and input information

  • Can be concatenated (added with other string variables)

  • Char variables (char stands for character) each hold one character, e.g. "#" "r" "A" "*" " . "

  • Useful for when a programmer wants to read one button press e.g. on a Game controlled keyboard

  • Strings can hold more than 1 character, but chars can't hold more than 1 character

Why are variables so useful?

  • Useful to keep track of things such as a users name or score

    • using variables, you can store this information to reference, add to, or modify

  • Other important uses for variables:

    • Taking input from a user
    • Making your program have variability (have it change based on certain factors)

Part 7: What are Conditional Statements? 31:54

  • Depending on certain conditions, we want code to do different things.

  • The most basic conditional statement is the If Statement

    • If something is True, do this, otherwise do something else

  • Brackets are used in most programming languages for If statements.

  • Each condition is evaluated as a boolean (true or false)

  • The Else-if statement will only be evaluated if the preceding if (or if-else) statement was bypassed due to it being false.

  • The else statement comes after an if/if-else statement and will always carry out instructions.

    • Carries out instructions if all previous cases are not true.

  • If we end up with too many else/if statements, we will need a Switch Statement:

    • An easily collapsible way to write any if-else statements.

      • Switch statements start with a colon (:)

    • By determining which 'Cases' that variable could be, each switch statement includes a default case (Else Statement)

    • Switch Statements must include the default case, hence the name 'default'

  • By having conditional statements, we add variability to our programming.

    • Having the program run differently based on user input
    • We want the program to adapt, when the user does something
    • without conditional statements, the software would just run the same way always

Part 9: What are loops? 44:26

  • How can we have code repeat segments without repeating lines of code? Loops.

  • A loop is a statement that is used to run certain instructions repeatedly

  • Instead of copy pasting lines of code 100 times, we could just create a loop statement

    The 3 types of loops are:

  • The For Loop, used when you want to carry out an instruction numerous times

    • Consists of an integer value, a conditional statement and an operation to modify the integer value after the instructions are completed.

  • To avoid infinite loops, we have to make sure that the given initial integer value and the operation will be met.

  • The For Each Loop, used for iterating through entire arrays or lists.

    • It will go through each element in the array and carry out instructions for each value.

  • The While Loop, which will continually carry out its operation while a conditional statement given to it is true.

  • The Do While Loop, functionally similar to while loops but will always carry out instructions at least once.

    • Instructions inside the loop will run once before checking the conditional statement.

Part 10: What are Errors? 49:37

  • What happens when our code doesn't work the way that we want it to? Errors abound!

  • Code doesn't always work as expected or as intended

  • Three types of errors:

    • Syntax errors:

      • Usually the easiest to solve. These consists of parts in your program where you fail to meet the programming rules. IDE's usually provide hints to your errors.

  • Runtime errors: Don't appear until you run the code. Caused by a part of your code not being able to be computed in a reasonable amount of time. e.g. infinite loops.

  • These happen when the computer is given some condition with no feasable way to finish the task.

  • These can be avoided by planning out your code before you being writing it

     

  • Logic errors happen when the code runs smoothly without runtime or syntax errors but the result isn't what you wanted. Often the hardest type of error to solve. Testing incrementally can reduce this.

Part 11: How do we debug code? 55:22

  • How do we fix syntax, runtime and logic errors? Debugging!

  • First step is to read the error message

  • If you don't understand it, visit an online forum or search online for a solution.

  • For Syntax or Runtime errors, the fix should be fairly easy.

  • Logic errors can use print statements to find where the code is going wrong.

  • Breakpoints pause the program when the line you placed the break point has been reached.

    • Used to determine where logic errors are occuring.

  • When you've found the error in your code, try commenting it out.

    • Comments serve as a documentation tool, but can also be used for debugging.

  • Backing up code frequently can help in preventing errors.

  • Version managers such as Git or Subversion help with this.

  • Run your program frequently, thus preventing you from saving a backup that doesn't work.

Part 12: What are Functions? 1:00:25

  • How can we reuse code to make our program more space-efficient and easy to read? Functions.
  • A function is a reusable sequence of statements designed to do a particular job.
  • Print statements, for loops and basic math operators are all functions.
  • A function is a segment of code that can be easily run by calling the function name. Depending on the function, my do something in return.
  • Simply calling on a piece of code whenever it is needed?
  • Functions serve many purposes. Used to recycle sections of code.
  • Functions are super powerful
  • There are thousands of functions, and often times you can just import the ones that you need, from libraries.

There are FOUR types of functions in programming.

  • Ones that are separated by whether or not they take in arguments

    • Arguments are variables that we pass into the function in order to be manipulated. Arguments can be strings, integers, arrays.
    • Functions can either return values or not. Returning something, simply means the function will return back to the user some form of information. Returning functions must be returned into something.

from learncpp.com: An argument is a value that is passed from the caller to the function when a function call is made. Note that multiple arguments are also separated by commas.

  • Ones that are separated by whether or not they return values.

    • Functions without arguments
    • No Return functions and arguments are used just to condense code, e.g. printStats():

  • Functions are super useful for making large changes to your code easily.
